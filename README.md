# Handoff: Ailter — AI 콘텐츠 신뢰 판단 모바일 앱

## Overview
Ailter는 SNS(Instagram, TikTok, YouTube Shorts) 콘텐츠가 AI로 생성·조작되었는지 실시간으로 감지하고 신뢰도를 알려주는 모바일 앱입니다. 사용자는 콘텐츠 분석 결과를 확인하고, 과거 기록을 보고, AI 콘텐츠 식별법을 학습하며, 보호 모드를 설정할 수 있습니다.

이 번들은 다음 화면을 포함합니다:
- **메인 4탭**: 홈(실시간 분석) · 기록 · 학습 · 설정
- **상세 페이지 2개**: 콘텐츠 신뢰도 상세, 최근 분석 기록 상세
- **디자인 시스템 문서**: 타이포·컬러·간격·elevation·UI Kit

## About the Design Files
이 번들의 파일은 **HTML로 만든 디자인 레퍼런스**입니다 — 의도한 외형과 동작을 보여주는 프로토타입이며, 그대로 복사해 출시할 프로덕션 코드가 아닙니다.

목표는 이 HTML 디자인을 **대상 코드베이스의 기존 환경(React Native, Flutter, SwiftUI, Jetpack Compose 등)에 그 환경의 패턴·라이브러리로 재현**하는 것입니다. 아직 코드베이스가 없다면, 모바일 앱에 가장 적합한 프레임워크를 선택해 구현하세요. 이 앱은 **모바일 네이티브 앱**이므로 React Native 또는 Flutter가 자연스러운 선택입니다.

`tokens.css`의 CSS 변수는 그대로 대상 환경의 테마/디자인 토큰(예: RN의 theme 객체, Flutter의 ThemeData, SwiftUI의 Color/Font 확장)으로 옮기면 됩니다.

## Fidelity
**High-fidelity (hifi)** — 최종 색상, 타이포그래피, 간격, 인터랙션이 확정된 픽셀 단위 목업입니다. 모든 화면은 첨부된 Figma 디자인(B-0 홈, C-0 기록, D-0 학습, E-0 설정)을 1:1로 재현했습니다. 개발자는 대상 코드베이스의 컴포넌트로 UI를 픽셀 단위로 재현해야 합니다.

기준 캔버스: **모바일 세로, 본문 폭 ~390px** (디자인 시안 기준). 프로토타입은 414px 폭 기기 프레임 안에 렌더됩니다.

### Typography
- **본문 폰트**: Pretendard (Variable). 브랜드 워드마크: Wanted Sans. 숫자/라벨: Inter.
- **공통**: 자간 `-0.02em` (−2%), 행간 1.5 (헤드라인 일부 1.0)
- 스케일:
  | 이름 | size / weight | 용도 |
  |---|---|---|
  | H1 | 40 / 700 | 디자인시스템 헤드라인 |
  | H2 | 24 / 700 | 인사 카드 제목, 섹션 대제목 |
  | H3 | 20 / 600 | 카드 제목(현재 콘텐츠 분석 결과) |
  | Subtitle | 18 / 700 | 분석 결과 문장, 섹션 소제목 |
  | Body1 | 16 / 400 | 본문, 리스트 설명, 설정 항목 |
  | Body2 | 14 / 400 | 학습 카드 설명, 캡션 보조 |
  | Caption | 13 / 400 | 시각(2분전), 뱃지, 상태 라벨 |

### Spacing (4px base)
`4 · 8 · 12 · 16 · 20 · 24 · 32 · 40`. 카드 내부 패딩 16px, 섹션 간 간격 24px, 본문 좌우 여백 20px가 기본 리듬.

### Radius
`sm 8px`(칩/인풋/리스트 썸네일) · `md 12px`(카드/버튼) · `lg 16px`(히어로/시트) · `full 999px`(뱃지/토글/상태점).

### Elevation
- `--elev-1`: `0 0 20px rgba(0,0,0,0.02)` — 기본 카드 (대부분의 카드)
- `--elev-2`: `0 4px 16px rgba(0,0,0,0.06)` — 강조
- `--elev-3`: `0 8px 24px rgba(0,0,0,0.12)` — 플로팅/오버레이
대부분의 카드는 `--elev-1` + `1px solid` 보더 조합을 씁니다.

## Screens / Views

### 공통 — App Shell
- **Status bar** (54px): 시간 9:41 + 신호/와이파이/배터리 아이콘. 네이티브에서는 OS 상태바로 대체.
- **App bar** (상단): 좌측 브랜드 워드마크 "Ailter"(Wanted Sans, 800, 레드, 자간 −4%), 우측에 "AI감지" 라벨 + 토글 + 알림 벨 + 프로필 아이콘.
  - **AI감지 토글**: 48×26 pill. ON = `--primary` 솔리드 배경 + 흰 노브 우측. OFF = `--gray-300` + 노브 좌측. 0.25s ease 전환.
- **Tab bar** (하단): 4탭 — 홈/기록/학습/설정. 활성 탭은 `--primary` 색 + weight 600, 비활성은 `--gray-400`. 각 탭 아이콘은 24px 라인 아이콘.
- **Home indicator**: 하단 140×5 검정 바.

### 1. 홈 (실시간 분석 화면) — B-0
사용자가 현재 보고 있는 콘텐츠의 실시간 분석 상태와 신뢰도를 확인.

- **인사 카드** (`.greet-card`, radius 12, padding 20, min-height 145):
  - **AI감지 ON 상태**: 배경 `linear-gradient(159deg, #FFF8F8 28%, #FFD8D7 135%)`, 보더 `1px #FFD8D7`. 상단에 레드 점(8px) + "AI 감지 활성화"(13px, `--primary`). 제목 "실시간 분석중..."(24/700, `--black`). 본문 "수현님, 안녕하세요!\n현재 콘텐츠를 분석중입니다"(16, `--gray-500`). 우측에 마스코트 이미지(`assets/mascot-greet.png`, ~140px).
  - **AI감지 OFF 상태**: 흰 배경, 보더 `1px #E9EBEF`. 회색 점(`--gray-400`) + "AI 감지 비활성화"(회색). 제목 "실시간 분석하기". 본문 "수현님, 안녕하세요!\n오늘도 분석을 시작할까요?". 같은 마스코트.
  - 토글 상태에 따라 텍스트/배경/점 색이 함께 전환됨.
- **현재 콘텐츠 분석 결과 카드** (흰 카드, `--elev-1`):
  - 헤더: "현재 콘텐츠 분석 결과"(20/700) + 우측 뱃지 "AI 생성 의심"(레드, `--primary-100` 배경).
  - 분석 항목 행: 48px 회색 썸네일(얼굴 스캔 아이콘) + "AI 감지 활성화"(13, subtle) + "얼굴 패턴이 비정상적으로 반복됩니다"(16/600).
  - **콘텐츠 신뢰도 박스** (`.gauge-box`, 회색 `#F9FAFB` 배경 + 보더, radius 12): "콘텐츠 신뢰도"(16/700) + chevron. **탭하면 → 콘텐츠 신뢰도 상세 페이지로 이동.** 반원 게이지(220×130, 42% 위치, 그라데이션 `--primary-300`→`--primary-500`) + 중앙 "주의" + "42%". 하단 범례(위험/주의/안전, 세로 구분선).
- **최근 분석 기록 섹션**: 헤더 "최근 분석 기록"(20/700) + "전체보기"(14, subtle). **"전체보기" 탭하면 → 최근 분석 기록 상세로 이동.** 카드 안 3행: 플랫폼 로고(44px) + 이름 + 시각 + 우측 태그 칩(사람 생성/딥페이크 의심/일반 콘텐츠).
- **경고 배너** (`.alert`): 레드 경고 아이콘 + "공유 전 다시 확인이 필요합니다" + 설명.
- **AI 콘텐츠 구별 팁**: 2열 그리드 타일. 연한 레드(`#FFF8F8`) 아이콘 박스(48px) + 제목 2줄.

### 2. 기록 (기록 메인 화면) — C-0
누적 분석 통계와 전체 기록 열람.

### 상세 A. 콘텐츠 신뢰도 상세 (홈 → 콘텐츠 신뢰도 박스)
- 헤더: ← 뒤로 + "콘텐츠 신뢰도"(중앙 정렬).
- 신뢰 등급 영역: 반원 게이지(42% 주의) + 범례 + "주의가 필요한 수준" 안내.
- **분석 항목별 신뢰도** 카드: 항목명 + 상태·% + 진행 바(색상=상태). 얼굴 패턴 일관성(위험 35%), 피부 질감 자연도(주의 48%), 배경 왜곡도(주의 60%), 메타데이터 정합성(안전 78%).
- 종합 의견 박스(`--primary-100` 배경) + 하단 액션 버튼 2개(신고하기 ghost / 분석 리포트 저장 primary).

### 상세 B. 최근 분석 기록 상세 (홈 → 전체보기)
- 헤더: ← 뒤로 + "최근 분석 기록".
- 검색바 + "검색" 버튼.
- 필터 칩: 전체(활성=레드) · AI 의심 · 일반 콘텐츠 · 위험 콘텐츠 (가로 스크롤).
- "이번 주 분석" 섹션: 평균 신뢰도 88점 + 주간 라인 차트(월~일, 면적+라인+점).
- "위험 콘텐츠 경고" 카드.
- "분석 기록" 리스트(C-0과 동일 행 구조, 썸네일 + 우상단 뱃지).

## Interactions & Behavior
- **탭 전환**: 하단 탭 클릭 시 해당 뷰만 표시(나머지 숨김), 활성 탭 색 변경. (`Ailter App.html`의 `.tab[data-tab]` → `.body[data-view]` 매칭 로직 참고.)
- **AI감지 토글**: 클릭 시 `on` 상태 토글 → 홈 인사 카드의 텍스트/배경/상태점 색 동시 전환.
- **상세 페이지 진입**: 홈에서 콘텐츠 신뢰도 박스(`[data-open="trust"]`) 또는 전체보기(`[data-open="records"]`) 클릭 → 해당 상세가 슬라이드인(translateY 12px, 0.32s cubic-bezier(.32,.72,0,1)). 뒤로가기(`[data-close]`)로 복귀. 네이티브에서는 표준 화면 push/pop 전환으로 구현 권장.
- **필터 칩**: 단일 선택, 활성 칩만 레드 배경.
- **게이지**: 반원 path를 따라 knob을 백분율 위치(0~1)에 배치. 구현 시 `getPointAtLength` 또는 동등한 각도 계산 사용.
- **가로 스크롤**(사례 학습, 필터 칩): 풀블리드 가로 스크롤, 스크롤바 숨김.
- **신뢰도 구간**: 0–40 위험 / 41–70 주의 / 71–100 안전 → 레벨 라벨 색 결정.

## State Management
- `aiDetectionOn: boolean` — AI감지 토글. 홈 인사 카드 표시에 영향.
- `activeTab: 'home' | 'history' | 'learn' | 'settings'` — 현재 탭.
- `openDetail: null | 'trust' | 'records'` — 상세 페이지 오버레이.
- `recordFilter: '전체' | 'AI 의심' | '일반 콘텐츠' | '위험 콘텐츠'` — 기록 상세 필터.
- 실제 앱에서는 분석 결과/기록/통계/학습 콘텐츠가 API에서 fetch되어야 함(현재는 정적 더미 데이터).
